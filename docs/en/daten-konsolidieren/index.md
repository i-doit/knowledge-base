# Data Consolidation

!!! warning "Please create a complete backup before making any changes to an interface/import. If the result is not satisfactory, the backup can then be restored"

When you break down [IT documentation](../glossar.md), you always end up with data. This data comes in various forms (mostly text). It is collected somewhere and then prepared and stored in a structured way in _i-doit_. Furthermore, it can be reused somewhere else in a different context. In the following articles, we deal with this "somewhere". In short: Where does the data come from and where is it needed?

-   [CSV Data Import](csv-datenimport/index.md) — With CSV data import, you transfer tabular data from Microsoft Excel, OpenOffice, or LibreOffice Calc directly into i-doit.
-   [CSV Data Export](csv-datenexport.md) — CSV data export
-   [h-inventory](h-inventory.md) — With h-inventory, you automatically inventory hardware, software, and entire networks.
-   [JDisc Discovery](jdisc/index.md) — Configuration and use of JDisc Discovery with i-doit
-   [Identifying Objects During Imports](objekt-identifizieren-bei-importen.md) — During data import, i-doit identifies already existing objects based on specific attributes and updates them instead of creating duplicates.
