# Introduction

Are you getting started with i-doit and want to build your IT documentation the right way from the beginning? Here you will find the ideal starting point — from "What is i-doit?" to productive operation.

## For Beginners

Read these articles in order — they build on each other:

1. **[What is i-doit?](was-ist-i-doit.md)** — What can i-doit do, what is it used for, and how is it structured? An overview for everyone just getting to know i-doit.

2. **[Concepts and terminology](konzepte-und-begriffe.md)** — Objects, object types, categories, attributes, and relationships — the building blocks of i-doit explained simply. Once you understand these concepts, everything else falls into place.

3. **[How do I start documenting?](wie-beginne-ich-zu-dokumentieren.md)** — Practical guide: Clarify goals, choose an entry strategy, adopt existing data, and plan ongoing maintenance.

4. **[IT documentation checklist](checkliste-fuer-die-it-dokumentation.md)** — Step by step from planning through installation to productive use. All important points to check off.
