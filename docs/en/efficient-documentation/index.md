# Efficient Documentation

i-doit offers much more than a structured database with a graphical interface. Numerous features are available to help you document as quickly and easily as possible. On the following subpages, you will find tips and tricks for manual handling:

*   [List Editing](list-editing.md) — List editing is a function that enables the simultaneous and easy editing of multiple objects.
*   [Mass Change](mass-change.md) — Mass change is a function in i-doit that allows multiple objects to be changed in one step based on change templates.
*   [Duplicating Objects](duplicate-objects.md) — With the duplicate function, you quickly create a copy of an existing object in i-doit -- including the desired category contents.
*   [Templates](templates.md) — Using templates
*   [Attribute Validation and Mandatory Fields](attribute-validation-and-required-fields.md) — If you want to ensure the uniqueness of attributes or expect only defined values when maintaining certain fields, or want to make fields mandatory...
*   [Permission Management](permission-management/index.md) — Who is allowed to see and edit what in i-doit? This frequently asked question stems from the need to establish a permission concept for your own IT documentation.
*   [Search](search.md) — Targeted research in IT documentation is often the fastest way to reach your goal -- faster than navigating through menu structures.
*   [Object Lock](objektsperre.md) — Collaborative work on IT documentation also brings some pitfalls.

Want to go beyond manual documentation? Then take a look at [Automation and Integration](../automation-and-integration/index.md).
