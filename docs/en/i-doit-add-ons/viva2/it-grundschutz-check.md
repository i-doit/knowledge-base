# IT-Grundschutz-Check

In the "IT-Grundschutz-Check" category, all requirements that belong to the modules assigned to this object are listed. The requirements are sorted by layer, module, and requirement type. In addition to the requirement number and title, the people responsible for implementation, the implementation status, and the implementation date are displayed.

[![IT-Grundschutz-Check](../../assets/images/de/i-doit-add-ons/viva2/it-grundschutz-check/it-grundschutz-check-beispiel.png)](../../assets/images/de/i-doit-add-ons/viva2/it-grundschutz-check/it-grundschutz-check-beispiel.png){:target="_blank"}

Clicking the magnifying glass icon also shows the requirement text and the implementation description.

[![Anforderungstext](../../assets/images/de/i-doit-add-ons/viva2/it-grundschutz-check/anforderungstext-beispiel.png)](../../assets/images/de/i-doit-add-ons/viva2/it-grundschutz-check/anforderungstext-beispiel.png){:target="_blank"}

Clicking the pencil icon opens the edit form to document the implementation status of this requirement.

[![Anforderung bearbeiten](../../assets/images/de/i-doit-add-ons/viva2/it-grundschutz-check/anforderung-bearbeiten-beispiel.png)](../../assets/images/de/i-doit-add-ons/viva2/it-grundschutz-check/anforderung-bearbeiten-beispiel.png){:target="_blank"}

!!! note "In the "Responsibility" field, only entries that are listed in the contact assignment of the associated module can be added."

If all basic requirements of a module are implemented (implementation status "Yes"), the module is marked with "Basic protection achieved".

[![Zielzustand](../../assets/images/de/i-doit-add-ons/viva2/it-grundschutz-check/zielzustand-beispiel.png)](../../assets/images/de/i-doit-add-ons/viva2/it-grundschutz-check/zielzustand-beispiel.png){:target="_blank"}

Similarly, the module is marked with "Standard protection achieved" when all basic requirements are implemented and all standard requirements are documented as implemented or dispensable.
