Documenting with i-doit is not an end in itself. In the following articles, we show what information can be derived from the data and how it can be presented in a variety of ways:

*   [Report Manager](./report-manager.md) — Reporting is a suitable tool for condensing, preparing, and sharing data with third parties.
*   [Notifications](./benachrichtigungen.md) — Be proactively informed via email about important events and states in the IT documentation.
*   [CMDB Explorer](./cmdb-explorer/index.md) — The CMDB Explorer graphically displays [relationships between objects](../../auswertungen/../grundlagen/objekt-beziehungen.md). Various dependency views can be generated. This makes it possible to create targeted analyses.
*   [Rack View](./rack-ansicht.md) — i-doit includes a visualization for 19\" racks.
*   [IP Lists](./ip-listen.md) — Which IP addresses are assigned to which host and which are still available?
