# JDisc Profiles

!!! warning "Please create a complete backup before every change to an interface/import. If the result is not satisfactory, it can then be restored."

Under **Administration → Import and Interfaces → JDisc → JDisc Profiles**, you configure the import profiles that define how data from JDisc Discovery is transferred to the i-doit CMDB. With JDisc profiles, you can connect objects and their attributes from the JDisc database with the same objects in i-doit.

In a profile, you define, among other things, which JDisc device types are mapped to which i-doit object types, which categories should be populated during import, and how existing data should be handled (create, update, or skip). You can create multiple profiles for different import scenarios -- for example, one profile for servers and another for network devices. Detailed documentation can be found under [JDisc Discovery](../../../../_archiv/jdisc-discovery.md).
