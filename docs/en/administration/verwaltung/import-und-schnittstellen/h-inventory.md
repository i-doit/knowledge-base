# H-Inventory

Here you can configure and activate the object matching profiles for the h-Inventory import.

You can reach this setting under **Administration → Import and Interfaces → H-Inventory**. When importing hardware inventory data via h-Inventory, i-doit must be able to determine whether an imported object already exists in the CMDB or whether it should be created as new. The object matching profiles serve this purpose: you define here which criteria (e.g., hostname, serial number, or MAC address) i-doit uses to identify and assign existing objects. This way, you avoid duplicates during repeated imports. Use this page when you are setting up the h-Inventory import or want to adjust the matching behavior.

Detailed documentation can be found under [h-Inventory](../../../daten-konsolidieren/h-inventory.md).
