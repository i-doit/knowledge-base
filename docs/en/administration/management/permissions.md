# Permissions

Via permissions, you control which users may access which areas of i-doit. A practical example can be found on our [blog](https://www.i-doit.com/blog/das-rechtesystem-in-i-doit/).

## Permission assignment

Here you configure the permissions for individual i-doit users and groups. → [READ MORE](../../efficient-documentation/permission-management/index.md)

[![Permission assignment](../../assets/images/de/administration/verwaltung/berechtigungen/1-rs.png)](../../assets/images/de/administration/verwaltung/berechtigungen/1-rs.png)

## Resolve permission system errors

With this function, you reset the Admin user's permissions to the factory defaults. Use it if you have locked yourself out due to a misconfiguration.

[![Resolve permission system errors](../../assets/images/de/administration/verwaltung/berechtigungen/2-rs.png)](../../assets/images/de/administration/verwaltung/berechtigungen/2-rs.png)
