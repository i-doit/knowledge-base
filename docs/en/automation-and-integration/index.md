# Automation and Integration

!!! warning "Please create a complete backup before making any changes to an interface/import. If the result is not satisfactory, the backup can then be restored"

No one is an island... Especially when dealing with IT documentation/CMDB, you find yourself right in the middle of the IT department and naturally want to make your work as easy as possible. So why create users manually when there is an LDAP interface, and why not take advantage of automatic email notifications?

The interfaces described are already included in _i-doit_:

*   [Email (SMTP)](../automation-and-integration/e-mail.md) — i-doit automatically sends emails on certain events or status queries -- for example, for notifications or workflows.
*   [i-doit console utility](../automation-and-integration/cli/index.md) — Commands of the i-doit console utility
*   [Network Monitoring](../automation-and-integration/network-monitoring/index.md) — The network monitoring integration consists of two components that can be used independently of each other.
*   [Trouble Ticket System (TTS)](../automation-and-integration/service-desk/index.md) — The service desk is a central customer interface in many organizations
*   [SNMP](../automation-and-integration/snmp.md) — With the SNMP category, you read SNMP values from objects in real time and display them in i-doit.
*   [Task Scheduling & Cron Jobs](../automation-and-integration/task-scheduling-and-cron-jobs.md) — Via Windows Task Scheduling, you automate recurring i-doit tasks such as logbook archiving, search index updates, or...
