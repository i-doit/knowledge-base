# Object Type: Country

The **Country** object type forms the top level of the location hierarchy in i-doit. Countries contain cities, which in turn contain buildings – creating a complete geographical structure of the entire IT infrastructure.

## Usage

- **Location Hierarchy**: Start the location documentation with countries as the top level.
- **International Infrastructure**: Document IT assets across different countries in a structured and consistent manner.
- **Compliance**: Use country assignment for data protection and compliance requirements (e.g., GDPR data storage location).
