# Object Type: Service

The **Service** object type documents IT services that are provided to users or customers. A service groups all underlying components – servers, applications, databases – into a logical unit.

## Usage

- **Service Catalog**: Document all IT services with description, SLA, and responsible persons.
- **Component Linking**: Assign all technical components that make up the service.
- **Impact Analysis**: Use the service structure to analyze the impact of component failures.
- **Service Level Management**: Record SLAs and monitor service availability.
