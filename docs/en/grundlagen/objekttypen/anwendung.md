# Object Type: Application

The **Application** object type documents end-user applications such as web browsers, office suites, ERP systems, and other software. Application objects are linked to the servers or clients on which they are installed.

## Usage

- **Software Inventory**: Record all installed applications in the organization as standalone objects.
- **License Management**: Link applications with license objects for complete license compliance.
- **Dependencies**: Document on which systems an application runs and which services it provides.
- **Service Mapping**: Assign applications to an IT service to link the technical and business perspective.
- **Variants**: Maintain different versions of the same application as variants of an application object.
