# Object list

The object lists in i-doit serve not only for the mere listing of [objects](../struktur-it-dokumentation.md), but also provide a variety of functions and can be almost completely customized.

[![object-list](../../assets/images/de/grundlagen/objekt-listen/1-ol.png)](../../assets/images/de/grundlagen/objekt-listen/1-ol.png)

Per [object type](../struktur-it-dokumentation.md), all assigned objects are displayed in a tabular format in the object list. Each row features an object with its [attributes](../struktur-it-dokumentation.md). Which attributes are displayed can be [configured](../objekt-liste/listenansicht-konfigurieren.md).

Columns can be sorted in ascending or descending order by clicking on the column headers. A [pre-sorting](../objekt-liste/listenansicht-konfigurieren.md) can be configured for this. Additionally, column widths can be varied per user.

[![object-list-2](../../assets/images/de/grundlagen/objekt-listen/2-ol.gif)](../../assets/images/de/grundlagen/objekt-listen/2-ol.gif)

Via the hatched area in each row, you assign an object to another object type:
You drag the object via drag-and-drop to the desired object type in the left menu tree, the **Object view**.

[![object-list-3](../../assets/images/de/grundlagen/objekt-listen/3-ol.gif)](../../assets/images/de/grundlagen/objekt-listen/3-ol.gif)

For this, the [interface display](../../administration/verwaltung/mandanten-name-verwaltung/einstellungen-mandanten-name.md#oberflachendarstellung) Drag 'n' Drop of objects must be set to Yes.

The following articles describe which additional functions and customizations are available for the object list:

-   **[Action bar](aktionsleiste.md)** In the upper area, the individual buttons for performing various actions are located. To the right of these buttons, a drop-down menu for switching the [condition](../lebens-und-dokumentationszyklus.md) can be found.
-   **[Navigating and filtering](navigieren-und-filtern.md)** To avoid losing track in long lists, you can navigate through multiple pages and filter the list by attributes.
-   **[Configuring list view](listenansicht-konfigurieren.md)** Each object list can be customized per user. Default settings for existing and new users can also be defined.
-   **[Advanced settings](erweiterte-einstellungen.md)** Getting the last bit out of the object lists? No problem.
