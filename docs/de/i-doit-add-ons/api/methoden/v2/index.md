Die neue Struktur ist viel strenger und erlaubt eine bessere Validierung der übergebenen Parameter und es ist nun möglich, deutlich strukturiertere und einfachere Endpunkte zu erstellen. Die neue Struktur wird einen neuen Service verwenden, d.h. die v2 API wird eine interne (i-doit Kern) v2 API verwenden.

Dies soll sicherstellen, dass die v2 API keine spezielle "eigene" Logik mehr implementiert. Die v2 API dient lediglich als Schnittstelle.

!!! example "Eine Dokumentation wird von der API selbst bereitgestellt oder ist in i-doit ersichtlich"

    === "Request system.endpoints.read.v2"

        ```json
        {
        "version": "2.0",
        "method": "system.endpoints.read.v2",
        "params": {
            "apikey": "KEY",
            "language": "en"
        },
        "id": 1
        }
        ```
