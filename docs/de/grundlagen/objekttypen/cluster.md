# Objekttyp: Cluster

Der Objekttyp **Cluster** dokumentiert hochverfügbare Systemverbünde. Ein Cluster fasst mehrere physikalische oder virtuelle Server zusammen, die gemeinsam Dienste bereitstellen und bei Ausfall eines Knotens automatisch Failover durchführen.

## Verwendung

- **Cluster-Topologie**: Dokumentiere alle Cluster-Knoten und ihre Rollen (aktiv/passiv).
- **Freigegebene Ressourcen**: Erfasse gemeinsam genutzte Speicher und Netzwerkressourcen des Clusters.
- **Clusterdienste**: Verknüpfe Clusterdienste, die im Cluster betrieben werden.
- **Failover-Analyse**: Nutze die Cluster-Dokumentation für Notfallpläne und Impact-Analysen.
